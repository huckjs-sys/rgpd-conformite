# Security Policy

## Supported Versions

Only the latest minor release receives security fixes.

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security issue in the **rgpd-conformite** plugin, please **do not open a public GitHub issue**. Because this plugin handles personal data subject to the GDPR, the impact of a leak is potentially significant.

Instead, report it privately:

1. Open a [private security advisory](https://github.com/huckjs-sys/rgpd-conformite/security/advisories/new) on GitHub, or
2. Email the maintainer at `huckjs@gmail.com` with the subject prefix `[SECURITY] rgpd-conformite`.

Include:

- A clear description of the vulnerability and its impact
- Steps to reproduce
- The plugin version and ChurchCRM version affected
- Any proof-of-concept code or screenshots, if available

## Response Timeline

- **Acknowledgement**: within 5 business days
- **Initial assessment**: within 10 business days
- **Fix and disclosure**: coordinated with the reporter; typically within 30 days for high-severity issues

## Disclosure

Once a fix is released, the advisory is published on GitHub. Credit is given to the reporter unless anonymity is requested.

## Scope

This policy covers the rgpd-conformite plugin code only. Vulnerabilities in ChurchCRM core should be reported to the ChurchCRM project: https://github.com/ChurchCRM/CRM/security
